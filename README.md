# Travel Route Assistant

旅行プランナーが交通手段・料金・ルートを調べる時間を削減するための MVP です。
AI はプランを作りません。区間ごとの移動手段・時間・料金を整理して提示し、
最終判断は常にユーザーが行います。

## セットアップ

```bash
npm install
cp .env.local.example .env.local
# .env.local に Google Maps の API キーを設定してください
npm run dev
```

http://localhost:3000 で確認できます。

## 必要な Google Cloud API キー

このアプリは 2 種類のキーを使い分けています（.env.local.example 参照）。

| 変数名 | 用途 | 有効化する API |
|---|---|---|
| `GOOGLE_MAPS_SERVER_API_KEY` | サーバー側から Directions API / Distance Matrix API を呼び出し、区間ごとの所要時間・距離・ルートや、最短ルート計算用の全地点間の移動時間を取得 | Directions API, Distance Matrix API |
| `NEXT_PUBLIC_GOOGLE_MAPS_API_KEY` | 地図サムネイル（Static Maps API）とフルスクリーン表示・最短ルートタブの地図（Maps Embed API） | Maps Static API, Maps Embed API |

`NEXT_PUBLIC_GOOGLE_MAPS_API_KEY` はブラウザに露出するため、Google Cloud Console で
**HTTP リファラー制限**をかけてください。`GOOGLE_MAPS_SERVER_API_KEY` はサーバーのみ
で使うため、**IP アドレス制限**を推奨します。

## 主な仕様と実装メモ

- **区間ごとの検索**: 入力順に隣り合う場所同士を Directions API で検索します。
- **交通手段の自動整理** (`lib/transportLogic.ts`): 徒歩の所要時間と走行距離をもとに、
  表示する交通手段を絞り込みます（徒歩15分以内→徒歩のみ、など）。長距離でも
  電車・車・飛行機は排他的にせず、比較できるようまとめて表示します。
- **車のルート（高速 / 下道）**: 「車」は高速道路を使う既定ルートを最大3件
  （Directions API の `alternatives=true`）、「車（下道）」は `avoid=tolls` で
  取得した迂回ルートを1件表示します。高速料金は概算（NEXCO平均レートベース）を
  燃料費に加算しています。
- **ルート詳細**: 電車は乗換駅・路線名・停車駅数、車は各ステップの案内文（Google の
  `route.summary` に含まれる高速道路名も表示）を「ルート詳細を見る」から確認できます。
- **料金の扱い**: 電車・タクシー・車の料金は、Google が返す fare 情報がない場合
  概算値を計算しています (`lib/mapsUtils.ts` の `estimateTaxiFareJPY` /
  `estimateTrainFareJPY` / `estimateHighwayTollJPY`)。Google Directions API の
  transit（電車）モードは日本の新幹線・特急など長距離路線のデータをほとんど
  持っていないため、取得できない場合は距離ベースの概算所要時間・料金を表示します
  （画面上に概算である旨を明記）。
- **飛行機**: 専用の運賃・時刻 API を組み込んでいないため、直線距離から概算所要時間
  のみを表示し、実際の料金確認は外部サイトに誘導します。
- **検索履歴・お気に入り**: 検索するたびに自動でブラウザの localStorage に履歴を
  保存し、右上の「履歴」から呼び出せます。各交通手段カードの★ボタンで、特定の
  区間・交通手段だけを「お気に入り」として個別に保存できます（バックエンドDBは
  使わず、ブラウザのlocalStorageのみで完結する簡易実装です）。
- **時間表記**: 60分以上は「2時間54分」のように時間・分で表示します。
- **通貨表示（言語連動）**: 表示言語に応じて金額の表示通貨を自動で切り替えます
  （日本語→円、English→ドル、Español/Català→ユーロ）。裏側の計算は常に円（JPY）で
  行い、表示の瞬間だけ **Frankfurter API**（欧州中央銀行の公式レートを毎日更新、
  APIキー不要・無料・無制限）から取得した最新レートで換算しています。レートは
  `/api/exchange-rates` 経由で1時間キャッシュされます。並び替え（最安・最速）や
  お気に入り・AIアシストの計算は常に円ベースで行われるため、表示言語を変えても
  判定結果は変わりません。
- **長距離電車の概算表示**: 実データが取得できない場合、「Google Mapsで乗換を調べる」
  ボタンから Google マップの乗換検索（世界中どこからでもアクセス可能）を直接開けます。
- **電車の複数候補**: Googleが実際に電車ルートを返せる区間では、車と同様に
  `alternatives=true` で最大3件の候補をアプリ内に表示します。Google側がそもそも
  ルートを一件も返せない区間（新幹線を含む長距離が多い）でのみ、概算1件＋
  「Google Mapsで乗換を調べる」ボタンにフォールバックします（実在しない駅名・
  路線を作り出すことはしません）。
- **最短ルート計算のタイムアウト対策**: Distance Matrix API の呼び出しに20秒のタイムアウトを
  設定しており、通信が固まった場合でも「計算中…」のまま無限に止まることはありません
  （タイムアウト時はエラー表示＋再試行ボタンが出ます）。また計算中は簡易的な進捗バーを表示します。
- **最短ルートタブの地点数制限への対応**: Distance Matrix API には「1リクエストあたり
  最大100件（出発地×到着地の組み合わせ）」という上限があり、11地点以上を一度に計算
  すると `MAX_ELEMENTS_EXCEEDED` エラーになっていました。`/api/distance-matrix` 側で
  自動的に複数のリクエストに分割してから結果を結合するようにし、地点数が多い場合でも
  動作するようにしました。
- **週末の混雑を考慮した最短ルート計算（任意）**: 各場所の入力欄にある📅アイコンを
  押すと、その場所を含む区間だけ、Google Distance Matrix API の `departure_time` /
  `traffic_model` パラメータを使い、直近の土曜正午ごろを想定した渋滞込みの参考値に
  差し替えて計算します（車移動のみ対応。電車・徒歩には影響しません）。フラグを付けて
  いない区間は今まで通り渋滞を考慮しない標準値のままです。順番の最適化自体が先に
  確定してから移動時間を計算する仕組みではないため、「その区間を実際に何時に通るか」
  までは厳密にシミュレートしておらず、あくまで「直近の土曜正午ごろ」という代表的な
  1時点を仮定した近似値である点にご留意ください。
- **「ルート検索」タブへの渋滞考慮の反映**: 📅マークを付けた場所を含む区間は、
  「ルート検索」タブの車（高速・下道・タクシー）の所要時間にも同じ仕組みで渋滞込みの
  参考値が反映されます。反映されたカードには **🚦 渋滞考慮** というバッジが表示され、
  ひと目で区別できるようにしています（バッジが無いカードは従来通りの標準値です）。
- **AI アシスト**: 外部 AI モデルを呼ばず、取得済みのルートデータ（最安・最速との差分）
  からルールベースで提案文を生成しています (`lib/aiSuggestions.ts`)。押したときだけ
  表示され、常に「最終判断はユーザー」という位置づけです。
- **多言語対応**: 日本語 / English / Español / Català。`lib/i18n.ts` に辞書があります。
- **「最短ルート」タブ**: 出発地点・最終地点は固定し、中間の場所のみを、移動時間が
  最小になる順番に並べ替える提案専用のタブです。「ルート検索」タブとは完全に独立して
  おり、交通手段の判断はこのタブでは行いません（あくまで巡回順の提案のみ）。
  - Google **Distance Matrix API** で全地点間の移動時間（車移動を目安）を一括取得
  - 出発地・終着地を固定しない「最近傍法＋2-opt」（決定的なアルゴリズムで、機械学習
    モデルではありません）で効率的な順番を計算
  - タブを開く（または場所を変更する）と自動的に再計算し、Maps Embed API の複数経由地
    ルートとして地図に表示
  - 「この順番を『ルート検索』タブに適用」ボタンを押した場合のみ、場所リストの順番が
    実際に変わります（自動適用はしません）

## Vercel へのデプロイ

1. GitHub にリポジトリを push
2. Vercel でプロジェクトを import
3. Environment Variables に `GOOGLE_MAPS_SERVER_API_KEY` と
   `NEXT_PUBLIC_GOOGLE_MAPS_API_KEY` を設定
4. Deploy
